# AntiGravity Chat Manager

**AntiGravity Chat Manager** is a standalone, deeply integrated Python desktop application designed to provide precise management of AntiGravity agent chat sessions on Windows operating systems. It enables you to index, package, restore, and deeply clean your AI chat environments with a beautiful CustomTkinter interface.

## 🎯 Features
- **Deep Indexing & Scanning:** Automatically locates and calculates accurate disk usage for all your past and active chat sessions.
- **Deep Clean:** Safely and permanently purges a chat session's entire data footprint from your disk, freeing up space.
- **Package Session:** Compresses an entire chat session environment into a single portable `.zip` backup archive.
- **Restore Session:** Extracts a packaged `.zip` backup directly into the active AntiGravity environment, seamlessly reviving the session for further use.
- **Standalone Executable:** Fully bundled into a single `AntiGravityChatManager.exe` file. Runs instantly on any Windows machine without requiring Python.

## 🔍 Data Architecture & Tracking Mechanisms
To properly manage a session without data loss or leftover artifacts, it is critical to understand exactly how AntiGravity stores its session data on Windows. 

AntiGravity utilizes the user's local AppData path to store session information. The root for all chat instances is:
`%USERPROFILE%\.gemini\antigravity\brain\`

Every individual chat session is allocated a unique folder based on its `conversation-id` (e.g., `31ac784a-a8a4-4d49-936f-4008a12a9dda`).

**Inside this unique `<conversation-id>` directory, the following data paths are deeply tracked and consumed by this manager:**
1. **`/.system_generated/logs/transcript.jsonl`** - The raw, chronological conversation history and tool-call execution logs.
2. **`/.system_generated/tasks/`** - Logs and outputs from any background tasks or asynchronous terminal commands executed by the agent.
3. **`/scratch/`** - Temporary files, ad-hoc scripts, and testing data generated by the agent during problem-solving.
4. **`/*.md` & Artifacts** - Persistent markdown documents like `implementation_plan.md`, `task.md`, `walkthrough.md`, and any user-facing code/files written explicitly into the session environment.

> **Crucial Note:** By targeting the root `<conversation-id>` folder, this application guarantees 100% capture of the session's footprint. The `Package` and `Clean` mechanisms operate recursively on this root folder, ensuring no orphaned files or hidden data remain anywhere else on the system. Global resources (like `knowledge/` items) remain untouched as they span multiple sessions.

## ⚙️ Operational Flow

### 1. Initialization (`scanner.py`)
When the application launches, the scanner engine resolves the exact local Windows environment variable for `%USERPROFILE%` and targets the `\.gemini\antigravity\brain\` directory. It iterates through every folder, identifying valid `conversation-id` directories. It reads recursive folder sizes and parses OS-level modification timestamps (favoring `transcript.jsonl` for accurate "last active" metrics). 

### 2. The Interface (`app.py`)
A `CustomTkinter` graphical user interface is constructed. It utilizes a dark-mode theme to align with premium development environments. The validated sessions are displayed in a dynamic, scrollable grid, surfacing exactly how much disk space each session is consuming.

### 3. Execution Engine (`manager.py`)
- **Packaging:** When triggered, the engine leverages Python's `shutil.make_archive` to recursively traverse the target `<conversation-id>` directory. It wraps all logs, scratches, and artifacts into an uncompressed or standard `.zip` file located in a user-chosen backup destination.
- **Cleaning:** To deeply clean, it leverages `shutil.rmtree` to forcefully and entirely purge the `<conversation-id>` node from the NTFS filesystem. 
- **Restoring:** Using `shutil.unpack_archive`, the engine takes a previously packaged `.zip` and carefully inflates it back into the `%USERPROFILE%\.gemini\antigravity\brain\` root, restoring the exact folder structure required for AntiGravity to recognize and resume the session.

## 🚀 Building from Source
If you wish to modify the code and rebuild the `.exe`:
1. Ensure Python 3 is installed.
2. Install dependencies: `pip install customtkinter pyinstaller`
3. Run `build.bat` in the project root.
4. The newly compiled standalone application will be generated in the `dist/` folder.
